# 삶의 부캐 (LIFE Alts) — 개발자 가이드

> 오늘 한 일이 캐릭터를 키우고, 삶에 따라 직업이 흐르듯 변하는 자기계발 RPG 앱.
> 이 문서는 **코드를 처음 보는 개발자**가 구조·데이터 흐름·확장 방법을 빠르게 파악하도록 돕습니다.
> (기획 의도는 `삶의부캐_기획서.md`, `삶의부캐_개발문서.md` 참고)

---

## 1. 한눈에

- **프레임워크**: React Native + TypeScript (Expo SDK 56)
- **저장**: 로컬 전용 (`AsyncStorage`). 서버 없음.
- **상태 관리**: 단일 React Context (`src/context/AppContext.tsx`). 외부 상태 라이브러리 없음.
- **화면 전환**: 라우터 없음. **단일 홈 화면 + 바텀시트 Modal 오버레이** 방식.
- **캐릭터**: `react-native-svg` 페이퍼돌(현재) / Rive 전환 준비됨(플래그 한 줄).
- **핵심 원칙**: 스탯·직업은 **저장하지 않고 `actions` 로그에서 매번 파생 계산**한다.

## 4. 핵심 개념 — 데이터 흐름

모든 것은 **하나의 로그(`actions`)** 에서 시작해 **두 갈래**로 갈라진다.

```
사용자 행동 (루틴 도장 / 퀘스트 완수 / 한 줄 일지)
        │
        ▼
   actions 로그에 append  ──  { ts, type, stat, amount, note, ... }
        │
   ┌────┴───────────────────────┐
   ▼ 즉시(그날)                  ▼ 누적(최근 14일)
 퀘스트 진행/완수               computeStats()  ← 감쇠 ×0.9/일
 스트릭 +1                      determineJob()  ← 70% 콤보 규칙
 마일스톤 금화 보상               │
 보상 토스트/축하 연출            직업 변경 시 → 전환 연출 + 변천사 기록
   └────┬───────────────────────┘
        ▼
   화면 표시 (HUD · 캐릭터 · 퀘스트 · 스탯)
```

### 왜 이렇게?
- **즉시 보상 ≠ 직업 변화**: 오늘 운동 한 번은 즉시 경험치를 주지만, 직업이 전사로 바뀌려면 2주간 누적돼야 한다. → "매일의 재미 + 장기적 정체성" 동시 충족.
- **파생 계산**: 스탯/직업은 저장하지 않는다. 앱을 열 때마다 `actions`를 다시 훑어 계산한다. → 판정 규칙을 바꿔도 과거 데이터가 깨지지 않는다.

## 5. 데이터 모델 (`src/types/index.ts`)

| 타입 | 설명 |
|---|---|
| `StatKey` | `str·int·cha·dex·wis·health` 6종 (고정) |
| `Action` | 한 일 기록. `stat`이 `'gold'`면 금화, `null`이면 순수 일지(스탯 무영향) |
| `Routine` | 반복 루틴 (활성/비활성). 매일 일일 퀘스트로 자동 생성됨 |
| `Quest` | `kind: 'daily' \| 'main'`. daily는 루틴에서 자동 생성, main은 길드에서 수동 생성 |
| `Character` | 파생 상태 (`stats`, `gold`, `job`) + 영속 상태 (`streak`, `longestStreak`, `lastActiveDate`, `cosmetics`) |
| `JobChangeEvent` | 직업 전환 이벤트 (변천사용, 별도 저장) |
| `Inventory` | 상점 소지품 (`spentGold`, `streakFreezes`, `ownedTitles`, `equippedTitle`) |

> **금화 이원화 주의**: `Character.gold`는 *감쇠된 활동 금화*로 **직업 판정 전용**(화면에 숫자로 안 보임). 사용자가 보는 **소지금**은 `AppContext.walletGold` = (평생 번 금화 합계, 감쇠 없음) − `inventory.spentGold`. **금화를 써도 직업은 안 바뀐다** (직업 = 행동, 잔고 아님).

## 6. 상태 관리 (`src/context/AppContext.tsx`)

앱의 모든 상태와 비즈니스 로직이 여기 모여 있다. 화면은 `useApp()` 훅으로 접근한다.

### 노출하는 값
- **상태**: `loading, actions, character, routines, quests, jobHistory, inventory, walletGold, todayDone`
- **일회성 신호**: `pendingJobChange, pendingReward, returnInfo, isFirstLaunch` (연출 트리거)
- **액션**: `recordRoutine, addRoutine, removeRoutine, toggleRoutineActive, completeQuest, abandonQuest, addMainQuest, addJournalNote, purchaseItem, equipTitle, useStreakFreeze, completeOnboarding, clearReward, clearJobChange, dismissReturn`

### 마운트 시 (로드 useEffect)
1. AsyncStorage에서 모든 데이터 병렬 로드
2. 온보딩 여부 확인 → 첫 실행이면 `isFirstLaunch`
3. 마지막 활동일과 오늘 비교 → 2일 이상 공백이면 `returnInfo` (복귀 환영)
4. `deriveCharacter()`로 캐릭터 재계산
5. 날짜가 바뀌었으면 일일 퀘스트 초기화, 활성 루틴 기준으로 없으면 자동 생성

### `recordRoutine` (가장 중요한 액션)
루틴 도장 한 번에 일어나는 일 (순서대로):
1. 루틴 action 생성
2. 일일/메인 퀘스트 진행도 갱신 (동기적으로 계산) + 메인 마일스톤 넘으면 보상 action 수집
3. 루틴 action + 보상 action들을 **한 번에** 원장에 저장
4. 스트릭 갱신 → `deriveCharacter()`로 재계산
5. 직업이 바뀌었으면 전환 연출 트리거 + 변천사 기록
6. 보상 토스트 표시 (마일스톤 보상 우선)

## 7. 직업 판정 엔진 (`src/utils/jobEngine.ts`)

앱의 "심장". 튜닝 상수가 파일 상단에 있다.

```
WINDOW_DAYS     = 14    최근 14일 actions만 사용
DECAY_RATE      = 0.9   하루 지날 때마다 가중치 ×0.9 (무게중심 이동)
COMBO_THRESHOLD = 0.7   2위 스탯이 1위의 70% 이상이면 조합 직업
GOLD_DOMINANCE  = 0.4   금화가 전체의 40% 이상이고 1위 스탯 이상이면 상인
```

- `computeStats(actions)` → 감쇠 가중 합으로 현재 스탯/금화 산출
- `determineJob(stats, gold)` → 직업 결정
  - 모두 0 → `novice`(견습생)
  - 6스탯 모두 있고 최대/최소 < 2배 → `adventurer`(숨은 직업)
  - 금화 지배 → `merchant`
  - 2위가 1위의 70%↑ → 조합 직업(`findComboJob`)
  - 그 외 → 1위 스탯의 단일 직업
- `deriveCharacter(actions, existing)` → 위 둘 + 영속 필드를 합쳐 `Character` 생성

> 직업/스탯 수치를 바꾸려면 이 파일의 상수만 조정하면 된다. 과거 데이터는 영향받지 않는다.

## 8. 화면 & 네비게이션 (`src/screens/HomeScreen.tsx`)

라우터가 없다. `tab` 상태값 하나가 어떤 오버레이를 띄울지 결정한다.

- **바텀 네비 4탭**: `home` / `stats` / `shop` / `guild` (모두 동일 스타일)
- **퀘스트 화면**은 네비 버튼이 아니라 **상단 "오늘의 퀘스트" 카드**(버튼)로 진입 → `tab='quest'`
- 각 오버레이는 `visible={tab === '...'}` 인 `Modal`. 닫으면 `tab='home'`.

### 상단 HUD
- 좌: 오늘의 퀘스트 카드 (탭 → 퀘스트 화면). 완료 표시는 **`todayDone` 기준**(오버레이와 일치)
- 중앙: 스트릭 배지 (탭 → 스탯)
- 우: 스탯 칩 + 소지금(`walletGold`)

### 진입 경로 요약
| 화면 | 진입 |
|---|---|
| 퀘스트(수행) | 상단 카드 |
| 스탯/연대기 | 스탯 탭, 스트릭 배지 |
| 상점 | 상점 탭 |
| 길드(생성) | 길드 탭 |

## 9. 캐릭터 렌더링 (`src/components/Character.tsx` 파사드)

앱 전체는 `Character` 한 컴포넌트만 쓴다. props로 **무엇을 표현할지**만 넘기고, 렌더 방식(SVG/Rive)은 숨긴다.

```tsx
<Character job={character.job} size={190} celebrate={allDone} mood={mood} />
```
- `job` → (Rive) `job_id` 입력 → 의상 전환
- `mood` → 표정: `idle`(평소) / `happy`(오늘 완료) / `sleepy`(스탯 잠듦) / `celebrate`(축하)
- `celebrate` → (SVG) 펄스 바운스 / (Rive) `is_success` 트리거

### Rive로 전환하려면 (`character/riveConfig.ts` 체크리스트)
1. `npx expo install rive-react-native` (dev build 필요)
2. `npx expo prebuild`
3. Rive 에디터에서 `character.riv` 제작 (상태머신 + `job_id`/`mood`/`is_success` 입력)
4. `assets/character.riv` 배치
5. `riveConfig.ts`의 `RIVE_ENABLED = true` + `Character.tsx`의 import/return 두 줄 주석 해제

→ `RiveCharacter.tsx`는 활성 전까지 어디서도 import하지 않아 **번들에 포함되지 않는다** (그래서 미설치 상태에서도 Expo Go가 안 깨짐).

## 10. 기능별 동작

| 기능 | 핵심 위치 | 동작 |
|---|---|---|
| 일일 퀘스트 도장 | QuestOverlay → `recordRoutine` | 라벨로 루틴을 찾아 기록. `todayDone`에 추가 |
| 메인 퀘스트 | GuildOverlay(생성) / QuestOverlay(완수) | 진행도/완수/포기. 마일스톤 보상 |
| 마일스톤 보상 | `utils/milestones.ts` | 25/50/75/100% → 금화 3/5/8/15. **보상도 action으로 적립** |
| 스탯 잠듦 | `jobEngine.getSlumberMessage` | 주 스탯 3일+ 미활동 → 졸린 표정 + "기운이 잠들었어요" |
| 스트릭/복구권 | `streak.ts` / `useStreakFreeze` | 연속일 +1, 공백 시 1로 리셋. 복구권으로 공백 메움 |
| 상점/금화 | ShopOverlay / `purchaseItem` | `walletGold`로 복구권·칭호 구매. 칭호는 홈 이름 앞 표시 |
| 한 줄 일지 | `addJournalNote` | `stat=null` action → 스탯 무영향, 연대기에 표시 |
| 직업 변천사 | `jobHistory` / StatsOverlay | 전환 이벤트를 별도 저장, 연대기에 타임라인 |
| 이탈 방지 | ReturnOverlay | 복귀 환영 + 최고 기록 박제 + 복구권 사용 |
| 살아있는 세계 | Background + `time.ts`(`useTimeOfDay`) | 기기 시각에 따라 배경 색 변화 (아침/낮/노을/밤). 1분마다 확인해 경계 넘으면 자동 전환 |

## 12. 확장 가이드

- **새 직업**: `types`의 `JobId`에 추가 → `data/jobs.ts`에 정의 → `jobEngine`의 `singleJob`/`findComboJob` 매핑 → `SvgCharacter`의 `JOB_PALETTE`/액세서리 → `voices.ts` 보이스 → (Rive 시) `riveConfig.JOB_ID_MAP`.
- **새 추천 루틴**: `data/defaultRoutines.ts`의 `ROUTINE_SUGGESTIONS`에 한 줄 추가.
- **새 상점 아이템**: `data/shopItems.ts`에 추가. `consumable`/`title` 외 종류는 `purchaseItem`에 분기 추가.
- **보이스 문구 수정**: `data/voices.ts` (직업×5상황).
- **판정 수치 튜닝**: `jobEngine.ts` 상단 상수.
- **시간대 색**: `utils/time.ts`의 `TIME_PALETTE`.

## 13. 코딩 컨벤션

- TypeScript strict. 화면은 함수 컴포넌트 + 훅.
- 비즈니스 로직은 전부 `AppContext` 또는 `utils/`에. 컴포넌트는 표시·입력만.
- 스타일은 각 파일 하단 `StyleSheet.create`.
- 색: 금색 강조 `#F0C070`, 배경 `#1E1E2E`/`#2A2A3E` 계열.
- 새 영속 데이터는 `storage.ts`에 `load*/save*` 쌍 + 기본값.
